# Nika — автономный AI-компаньон для Discord

Nika — экспериментальный Discord AI-бот на Python, использующий `discord.py`, SQLite-память и локальные LLM-модели (LM Studio / llama.cpp / OpenAI-compatible API).

В отличие от обычных Discord-ботов, которые реагируют только на команды или прямые упоминания, Nika проектируется как полуавтономный участник сообщества, способный наблюдать за общением, помнить пользователей и естественно участвовать в разговорах.

---

## Идея проекта

Цель проекта — создать AI-персонажа, который ощущается не как чат-бот-ассистент, а как живой участник Discord-сервера.

Nika должна уметь:

- наблюдать за разговорами на сервере
- понимать контекст общения
- помнить пользователей и взаимодействия с ними
- формировать собственные впечатления о людях
- естественно реагировать на живое общение
- самостоятельно подключаться к разговорам
- читать историю каналов и делать краткие сводки
- поддерживать долговременную социальную память
- избегать шаблонного поведения "бота-помощника"

Это не саппорт-бот.

Это попытка создать социально-осознанного AI-участника Discord.

---

## Концепция

Обычный Discord-бот работает так:

Пользователь → команда → ответ

Nika должна работать так:

Активность на сервере → наблюдение → анализ контекста → обновление памяти → автономная реакция

То есть бот:

- следит за активностью
- анализирует происходящее
- решает, стоит ли вмешиваться
- запоминает пользователей
- формирует историю взаимодействий
- ведёт себя как участник сообщества, а не как сервис

---

## Ключевые возможности

### Автономное участие в разговорах
Бот должен уметь определять активные беседы и при необходимости:

- поставить реакцию
- оставить короткий комментарий
- пошутить
- подколоть
- ответить по теме
- либо промолчать

С защитой от спама:

- cooldown между вмешательствами
- вероятностная логика реакции
- защита от самозацикливания
- эвристики активности каналов

---

### Глубокое чтение каналов
Вместо чтения только последних нескольких сообщений бот должен уметь:

- читать большие участки истории
- анализировать происходящее в канале
- продолжать чтение более старых сообщений
- понимать follow-up запросы

Примеры:

> что тут обсуждали?

> а до этого?

> а ещё?

> продолжай

---

### Summary Pass
Чтобы не гонять десятки сырых сообщений каждый раз:

1. Бот читает историю канала
2. Делает промежуточную AI-сводку
3. Сохраняет summary
4. Использует summary для follow-up вопросов

Это снижает:

- расход токенов
- нагрузку на модель
- задержки ответа

---

### Социальная память / карточки пользователей
Для каждого пользователя бот должен хранить профиль:

- user id
- username / display name
- краткое описание
- черты характера
- повторяющиеся интересы
- темы общения
- история взаимодействий
- уровень симпатии / affinity
- мнение бота
- заметки

Примеры:

- любит FPS игры
- часто шутит / троллит
- технически подкован
- активный участник сервера

---

### Контекст follow-up
Бот должен помнить предыдущие действия.

Пример:

Пользователь:
> что обсуждали в #general

Следом:
> а ещё?

Бот должен понимать, что речь всё ещё про тот же канал.

---

### Защита от тупых ответов
Система должна предотвращать:

- повтор сообщений пользователя
- повтор собственных ответов
- пустые ответы
- дегенеративные ответы

Типа:

> мм

> ок

> да

> повторение фразы пользователя

---

### Персона
Nika НЕ должна вести себя как служба поддержки.

Характер:

- живой
- разговорный
- с юмором
- Discord-native
- может иронизировать
- может молчать
- не навязывает помощь

Избегать:

- "Чем могу помочь?"
- "Я готов помочь"
- формального assistant-tone

---

## Архитектура

Основные модули:

- `discord_app.py` — Discord event loop / watcher
- `planner.py` — принятие решений
- `executor.py` — выполнение действий
- `direct_parser.py` — быстрый парсинг намерений
- `memory.py` — память / SQLite
- `prompts.py` — системные промпты / персона
- `llm.py` — интерфейс к модели
- `text_utils.py` — фильтрация / защита от повторов
